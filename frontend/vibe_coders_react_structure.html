Folder guide
assets/
Static media — images, icons, custom fonts.
components/
All reusable UI — split into common, layout, and ui sub-layers.
pages/
One file per route. Pages compose components, they don't define them.
routes/
Routing logic and guards — protected & admin access control.
services/
API calls and business logic — never mix these into components.
hooks/
Custom React hooks — reusable stateful logic extracted cleanly.
context/
Global state via React Context — auth, theme, user session.
redux/ (opt.)
Only add Redux when Context isn't enough. Store, slices, actions.
utils/
Pure helper functions — no React, no side effects. Fully testable.
styles/
Global CSS, design tokens, and theme variables.
config/
Env handling and Axios base configuration.
The rule of thumb
"Every file should have one reason to exist and one obvious place to live."
Structure now so you don't refactor painfully at 50 files.
