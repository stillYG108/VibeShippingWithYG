Folder guide
config/
DB connection, env loading, and CORS setup. Boot-time wiring, nothing else.
controllers/
Handle req → res. Call services, return JSON. No raw DB queries here.
models/
MongoDB/Mongoose schemas. Shape of your data. One file per collection.
routes/
API endpoints — map HTTP verbs to controller functions. Thin wiring only.
middleware/
Auth guards, global error handler, Multer file upload — runs between route and controller.
services/
Business logic and third-party integrations (email, payments). Keep controllers thin by putting the heavy work here.
utils/
Pure helper functions — JWT generator, async wrapper, reusable validators. No side effects.
validators/
Request body validation schemas (Joi / express-validator). Reject bad data before it hits the controller.
uploads/
Landing zone for user-uploaded files and images. Add to .gitignore — never commit binaries.
The rule of thumb
"Controllers orchestrate, services think, models remember."
Keep each layer dumb about the others — your future self will thank you at 3 am.
